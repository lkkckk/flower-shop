# 任务：实现商品管理模块（鲜花批发系统第一个业务功能）

## 背景

这是鲜花批发店管理系统的第一个完整业务模块。项目已经初始化完成，有 Nuxt 4 + Ant Design Vue + Prisma + SQLite 的基础骨架，有 default 和 pos 两套布局，Prisma schema 已经定义好所有表。

现在需要实现商品管理的完整 CRUD 功能，包括商品列表、新增、编辑、删除，以及每个商品的多单位换算配置。

## 技术栈约束

- 页面和组件：Vue 3 + <script setup lang="ts"> 语法
- UI：Ant Design Vue 4.x 组件
- 样式：Tailwind CSS
- API：Nuxt server/api 文件系统路由
- 数据库：Prisma（使用 server/utils/prisma.ts 的单例）
- 类型：从 @prisma/client 导入 Product 等类型
- 状态：如需全局状态用 Pinia，本模块主要用组件内 state

## 业务需求

商品是系统的基础数据，每个商品代表一种花材（如"玫瑰 A 级 70cm 红色"）。商品有以下特点：

1. **多单位销售**：每个商品有一个基础单位（如"枝"），可以配置多个销售单位（如"1 扎 = 10 枝"、"1 箱 = 200 枝"），开单时按不同单位卖
2. **分级分色**：同一种花有不同等级（A/B/C）、不同颜色、不同规格
3. **多档价格**：默认价、VIP 价、批发价
4. **花期管理**：每个商品有默认花期天数，入库时用来算到期日

## 具体任务

### 第 1 部分：后端 API

创建以下 5 个 API 文件：

**server/api/products/index.get.ts**
- GET /api/products
- 支持 query 参数：page（默认1）、pageSize（默认20）、keyword（搜索名称）、category、status
- 返回 { data: { list, total, page, pageSize }, error: null }
- list 中的每个商品包含其 unitConversions 关联数据（include）
- 按 updatedAt desc 排序

**server/api/products/index.post.ts**
- POST /api/products
- body 包含商品字段 + unitConversions 数组
- 用 prisma.$transaction 在一个事务里创建商品和所有单位换算
- 返回 { data: 创建的商品（含 unitConversions）, error: null }
- 出错时返回 { data: null, error: { message, code } }，HTTP 状态码 400

**server/api/products/[id].get.ts**
- GET /api/products/:id
- 返回商品详情含 unitConversions
- 不存在时 HTTP 404，error.code = 'NOT_FOUND'

**server/api/products/[id].put.ts**
- PUT /api/products/:id
- body 包含更新字段和 unitConversions 数组
- 用事务处理：更新商品 + 删除旧的 unitConversions + 创建新的 unitConversions（最简单的方式处理多对一关联更新）
- 返回更新后的商品

**server/api/products/[id].delete.ts**
- DELETE /api/products/:id
- 软删除：设置 status = 'inactive'（不是真的删除，因为订单明细还引用着）
- 返回 { data: { success: true }, error: null }

所有 API 都要：
- 用 defineEventHandler 包裹
- 用 readBody 读取 body
- 用 getQuery 读取 query
- 用 getRouterParam 读取路由参数
- 错误用 createError 抛出

### 第 2 部分：前端 composable

创建 composables/useProducts.ts，封装所有商品相关的 API 调用：

```typescript
// 期望的 API
const { 
  fetchProducts,      // (params) => Promise<{ list, total }>
  fetchProduct,       // (id) => Promise<Product>
  createProduct,      // (data) => Promise<Product>
  updateProduct,      // (id, data) => Promise<Product>
  deleteProduct,      // (id) => Promise<void>
  loading,            // Ref<boolean>
  error,              // Ref<string | null>
} = useProducts()
```

内部用 $fetch 调 API，统一处理 loading 状态和错误。失败时用 Ant Design Vue 的 message.error 提示用户。

### 第 3 部分：商品列表页

**pages/products/index.vue**

布局：
- 顶部工具栏：左侧搜索框（a-input-search，搜索名称）和分类筛选（a-select）、状态筛选，右侧"新增商品"按钮
- 中间主区域：a-table 展示商品列表
- 表格列：名称、分类、等级（a-tag 区分颜色 A=红 B=橙 C=灰）、规格、基础单位、默认价、VIP价、批发价、花期天数、状态、操作
- 操作列：编辑、删除两个按钮（a-button type="link" size="small"）
- 底部：分页控件（a-pagination）
- 手机端：表格横向滚动 + 固定前两列（名称、操作）
- 新增/编辑通过打开弹窗（a-modal），不要跳转新页面

交互：
- 搜索框防抖 300ms
- 筛选变化立即触发查询
- 点击"新增"打开空表单弹窗
- 点击"编辑"打开带数据的表单弹窗
- 点击"删除"用 a-popconfirm 二次确认，确认后调用 deleteProduct

状态：
- 用 ref 管理：products 列表、loading、pagination、searchKeyword、filters、modalVisible、editingProduct

### 第 4 部分：商品表单组件

**components/products/ProductForm.vue**

接受 props：
- visible: boolean（弹窗显示）
- product: Product | null（null 表示新增）

emit 事件：
- update:visible
- success（保存成功后触发，父组件刷新列表）

表单字段（用 a-form 分组）：

第一组：基本信息
- 商品名称（a-input，必填）
- 分类（a-select，选项：鲜切花、绿植、花束、配材、其他）
- 等级（a-select，选项：A级、B级、C级、不分级）
- 颜色（a-input，可选）
- 规格（a-input，占位符"70cm"）

第二组：价格设置
- 基础单位（a-input，必填，占位符"枝"）
- 默认价（a-input-number，必填，精度 2 位小数）
- VIP 价（a-input-number，可选）
- 批发价（a-input-number，可选）

第三组：库存相关
- 默认花期天数（a-input-number，必填，默认 7）

第四组：单位换算（这是难点，认真做）
- 标题"单位换算配置"
- 说明文字："配置如何从基础单位换算到销售单位。例如：基础单位是'枝'，可以配置'1 扎 = 10 枝'"
- 一个动态列表，每行两个输入：
  - 单位名称（a-input，如"扎"）
  - 对应基础单位数量（a-input-number，如 10）
- 右侧一个删除按钮
- 列表下方"+ 添加单位"按钮
- 至少允许 0 行（完全不配置换算也可以）

第五组：状态
- 状态（a-radio-group：在售、停售）
- 默认在售

操作：
- 底部"取消"和"保存"按钮
- 保存时先做前端校验，通过后调用 createProduct 或 updateProduct
- 成功后 emit('success') 并关闭弹窗
- 失败时错误会被 composable 里的 message.error 弹出，无需额外处理

表单校验规则：
- 必填字段不能为空
- 价格必须 >= 0
- 花期天数必须 >= 1
- 单位换算中每个单位名称不能重复（前端提前校验，避免触发 unique 约束报错）
- 单位换算中的数量必须 > 0

## 开发注意事项

1. **类型安全**：所有 API 响应、组件 props、表单字段都要有 TypeScript 类型
2. **错误处理**：所有 API 调用都要 try-catch，用 message.error 提示用户
3. **加载状态**：按钮在提交时要 :loading="loading"
4. **空状态**：表格无数据时显示 a-empty
5. **响应式**：表单在手机端单列显示，在桌面端可以两列
6. **不要用 Mock 数据**：所有数据都要真实从 API 来
7. **保存后刷新**：新增或编辑成功后自动刷新列表，不要让用户手动刷
8. **删除确认**：用 a-popconfirm 而不是 a-modal.confirm
9. **导入顺序**：先 import 类型（import type），再 import 值
10. **Nuxt 自动导入**：ref、computed、onMounted、useFetch 等不需要手动 import
