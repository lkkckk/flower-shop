# 任务：初始化一个鲜花批发店管理系统的 Nuxt 3 项目

## 项目背景

我要做一款鲜花批发店自用的进销存 + 收银系统。单店自用，部署在店里一台电脑上，店里手机平板通过局域网访问。核心功能包括商品管理、客户管理、库存批次管理、多单并行开单、欠款对账、报表。

这个任务是项目初始化阶段，目标是搭起一个能跑的空壳，后续业务功能在此基础上迭代。

## 技术栈（严格遵守，不要替换）

- 前端框架：Nuxt 3（最新稳定版）+ TypeScript
- UI 组件库：Ant Design Vue 4.x（通过 @ant-design-vue/nuxt 模块集成）
- 原子 CSS：Tailwind CSS（通过 @nuxtjs/tailwindcss 模块集成）
- 状态管理：Pinia（通过 @pinia/nuxt 模块集成）
- 后端：Nuxt server/api（不使用独立后端框架）
- ORM：Prisma 5.x
- 数据库：SQLite（本地文件 prisma/dev.db）
- 包管理：pnpm
- 图标：@ant-design/icons-vue

### 第 3 步：配置 Prisma + SQLite

初始化 Prisma，使用 SQLite 数据库。创建 prisma/schema.prisma，包含以下完整的数据模型：

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}

model Product {
  id              Int      @id @default(autoincrement())
  name            String
  category        String?
  baseUnit        String
  grade           String?
  color           String?
  specification   String?
  defaultPrice    Float
  vipPrice        Float?
  wholesalePrice  Float?
  shelfLifeDays   Int      @default(7)
  attributes      Json?
  status          String   @default("active")
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  unitConversions UnitConversion[]
  stockBatches    StockBatch[]
  orderItems      OrderItem[]
}

model UnitConversion {
  id          Int     @id @default(autoincrement())
  productId   Int
  fromUnit    String
  toBaseQty   Float
  product     Product @relation(fields: [productId], references: [id], onDelete: Cascade)

  @@unique([productId, fromUnit])
}

model StockBatch {
  id           Int      @id @default(autoincrement())
  productId    Int
  batchNo      String   @unique
  inboundDate  DateTime
  expiryDate   DateTime
  inboundQty   Float
  currentQty   Float
  costPrice    Float
  status       String   @default("in_stock")
  notes        String?
  createdAt    DateTime @default(now())

  product      Product         @relation(fields: [productId], references: [id])
  orderItems   OrderItem[]
  movements    StockMovement[]

  @@index([productId, status])
  @@index([expiryDate])
}

model StockMovement {
  id             Int      @id @default(autoincrement())
  batchId        Int
  type           String
  qtyChange      Float
  relatedOrderId Int?
  operator       String?
  notes          String?
  createdAt      DateTime @default(now())

  batch          StockBatch @relation(fields: [batchId], references: [id])

  @@index([createdAt])
}

model Customer {
  id          Int      @id @default(autoincrement())
  name        String
  phone       String?  @unique
  address     String?
  level       String   @default("normal")
  balance     Float    @default(0)
  totalOwed   Float    @default(0)
  notes       String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  orders      Order[]
  payments    Payment[]

  @@index([phone])
}

model Order {
  id              Int      @id @default(autoincrement())
  orderNo         String   @unique
  customerId      Int?
  orderType       String   @default("retail")
  totalAmount     Float
  paidAmount      Float    @default(0)
  owedAmount      Float    @default(0)
  paymentMethod   String?
  deliveryTime    DateTime?
  deliveryAddress String?
  notes           String?
  status          String   @default("pending")
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  customer        Customer?   @relation(fields: [customerId], references: [id])
  items           OrderItem[]
  payments        Payment[]

  @@index([createdAt])
  @@index([customerId])
}

model OrderItem {
  id          Int     @id @default(autoincrement())
  orderId     Int
  productId   Int
  batchId     Int
  unit        String
  qty         Float
  baseQty     Float
  unitPrice   Float
  subtotal    Float
  grade       String?
  color       String?
  notes       String?

  order       Order      @relation(fields: [orderId], references: [id], onDelete: Cascade)
  product     Product    @relation(fields: [productId], references: [id])
  batch       StockBatch @relation(fields: [batchId], references: [id])
}

model Payment {
  id              Int      @id @default(autoincrement())
  customerId      Int
  orderId         Int?
  amount          Float
  paymentMethod   String
  type            String
  notes           String?
  createdAt       DateTime @default(now())

  customer        Customer @relation(fields: [customerId], references: [id])
  order           Order?   @relation(fields: [orderId], references: [id])

  @@index([customerId, createdAt])
}
```

执行 `pnpm prisma migrate dev --name init` 生成初始迁移。

### 第 4 步：创建 Prisma Client 单例

在 server/utils/prisma.ts 创建 Prisma client 的单例，供所有 API 路由使用。注意处理开发环境下的热更新问题（避免创建多个 client 实例）。

### 第 5 步：创建两套布局

创建 layouts/default.vue（标准后台布局）：
- 使用 a-layout 组件
- 左侧 a-layout-sider 侧边栏，宽度 240px，可折叠
- 侧边栏菜单（a-menu）包含：首页(/)、商品管理(/products)、客户管理(/customers)、库存管理(/stocks)、开单收银(/pos)、订单记录(/orders)、对账单(/payments)、报表(/reports)、系统设置(/settings)
- 顶部 a-layout-header，左侧显示折叠按钮和面包屑，右侧显示临期提醒铃铛（a-badge 占位）和用户菜单
- 主内容区 a-layout-content 渲染 <slot />
- 响应式：屏幕宽度 <768px 时侧边栏变成抽屉，通过汉堡菜单按钮打开

创建 layouts/pos.vue（POS 收银端布局）：
- 没有侧边栏，最大化工作区
- 顶部紧凑栏：左侧显示"← 返回后台"按钮（点击跳转 /）、店铺名，右侧显示临期提醒铃铛和用户
- 顶部栏下方留给多单 Tab（此阶段只放占位符 <slot name="tabs" />，具体实现在产出 3）
- 主工作区 <slot />，占据剩余所有空间
- 布局固定不滚动，内部区域各自滚动
- 响应式：手机端顶部栏压缩，主工作区变成单列垂直流

两个布局都要支持手机、平板、桌面三端响应式。

## 必须遵守的规范

1. **所有代码使用 TypeScript**，不要写 JavaScript
2. **所有新文件顶部不写 import Vue 之类的**——Nuxt 3 自动导入
3. **组件名用 PascalCase**，文件名与组件名一致
4. **Prisma client 全局单例**，不要在每个 API 里 new PrismaClient()
5. **Ant Design Vue 组件按需引入**，不要全量引入以免首屏过大
6. **响应式断点统一使用 Tailwind 的约定**：sm 640 / md 768 / lg 1024 / xl 1280
7. **不要用 dayjs 的 locale 全局配置**，在需要的地方单独处理以免 SSR 出问题
8. **所有 API 路由返回统一格式**：{ data, error }，方便前端统一处理
